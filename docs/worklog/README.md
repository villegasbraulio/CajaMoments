# Implementation Worklogs

Chronological records of meaningful repository work.

## File convention

- Use one file per month: `YYYY-MM.md`.
- Copy `TEMPLATE.md` when the current month file does not exist.
- Append entries in chronological order.
- Keep each entry concise and factual, normally under about 50 lines.

## What belongs here

- Goal and scope inspected.
- Implementation summary and files changed.
- Exact verification commands and outcomes.
- Plan deviations.
- ADRs created or updated.
- Assumptions, reusable failures, unresolved risks, and handoff state.

## What does not belong here

- Every shell command or typo.
- Full logs.
- Long architectural rationale already captured in an ADR.
- Stale current-state information that belongs in `docs/PROJECT_STATE.md`.
